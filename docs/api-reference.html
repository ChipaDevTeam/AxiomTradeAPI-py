API Reference

This document provides a detailed reference for our trading bot API, allowing you to programmatically interact with our services and automate your trading strategies.

Authentication

All API requests require authentication. Please refer to the authentication guide for details on how to obtain and use your API keys.

Authenticate

HTTP Method: POST

Endpoint: Details depend on AxiomAuth class

Description: Authenticates the user and returns an API token for subsequent requests.

Parameters:

Name Type Description Required
username string Your API username. Yes
b64_password string The one-time password for 2FA. Yes

Response:

Returns a JSON object containing the API token and its expiration time.


{
  "token": "your_api_token",
  "expires_at": "timestamp"
}

Refresh Access Token

HTTP Method: POST (Details depend on AxiomAuth class)

Endpoint: Details depend on AxiomAuth class

Description: Refreshes the access token using a valid refresh token.

Parameters:

Name Type Description Required
Requires a valid refresh token set in the client instance. N/A N/A Yes

Response:

Returns the new access token as a string.


"new_access_token_string"

Error Codes:

  • 401 Unauthorized: Invalid or expired refresh token.
  • 400 Bad Request: No refresh token available.

Trading Operations

Place Order

HTTP Method: POST

Endpoint: Specific endpoint not available in client.py, likely a trading endpoint.

Description: Places a new trading order.

Parameters:

Name Type Description Required
Requires a valid access token for authentication. N/A N/A Yes
symbol string The trading symbol (e.g., "EURUSD", "BTCUSDT"). Yes
type string Order type (e.g., "market", "limit", "stop"). Yes
side string Order side ("buy" or "sell"). Yes
volume number The order volume. Yes
price number The order price (required for limit/stop orders). Optional
stop_loss number Stop loss price. Optional
take_profit number Take profit price. Optional

Response:

Returns a JSON object with the order details.


{
  "order_id": "unique_order_id",
  "symbol": "EURUSD",
  "type": "market",
  "side": "buy",
  "volume": 0.1,
  "status": "filled"
}

Error Codes:

  • 400 Bad Request: Invalid parameters provided.
  • 401 Unauthorized: Invalid or expired API token.
  • 400 Insufficient Funds: Not enough balance to place the order.

Data Retrieval

Get Historical Data

HTTP Method: GET

Endpoint: Specific endpoint not available in client.py, likely a data endpoint.

Description: Retrieves historical price data for a given symbol and timeframe.

Parameters:

Name Type Description Required
Requires a valid access token for authentication. N/A N/A Yes
symbol string The trading symbol (e.g., "EURUSD", "BTCUSDT"). Yes
timeframe string The data timeframe (e.g., "1m", "5m", "1h", "1d"). Yes
start_time timestamp The start time for the data range. Optional
end_time timestamp The end time for the data range. Optional
limit integer Maximum number of data points to return. Optional (default: 100)

Response:

Returns a JSON array of historical price data (OHLCV format).


[
  {
    "timestamp": "timestamp",
    "open": 1.1234,
    "high": 1.1235,
    "low": 1.1230,
    "close": 1.1233,
    "volume": 1000
  },
  {
    "timestamp": "timestamp",
    "open": 1.1233,
    "high": 1.1236,
    "low": 1.1231,
    "close": 1.1235,
    "volume": 1200
  }
]

Error Codes:

  • 400 Bad Request: Invalid parameters provided.
  • 401 Unauthorized: Invalid or expired API token.
